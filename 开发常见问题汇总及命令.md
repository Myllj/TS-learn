# tsc & nodemon 全栈开发使用教程（高手版）

## 一、tsc（TypeScript Compiler）：官方编译器

### 1. 核心定位

**TS 项目基石**：负责**类型检查** + **将 TS 编译为 JS**，仅存在于编译阶段，运行时消失。

### 2. 安装

```bash
# 本地安装（推荐，项目隔离）
npm install -D typescript

# 全局安装（方便命令行直接调用）
npm install -g typescript

# 验证
tsc -v
```



### 3. 初始化配置（tsconfig.json）

```bash
# 生成标准 tsconfig.json（必做）
tsc --init
```

**推荐基础配置（tsconfig.json）**：

```json
{
  "compilerOptions": {
    "target": "ES2020",       // 编译目标 JS 版本
    "module": "CommonJS",     // 模块系统
    "outDir": "./dist",       // 编译输出目录
    "rootDir": "./src",       // 源码根目录
    "strict": true,           // 开启严格模式
    "esModuleInterop": true,  // 兼容 CommonJS/ES 模块
    "skipLibCheck": true,     // 跳过库文件检查（提速）
    "sourceMap": true         // 生成 sourceMap（调试必备）
  },
  "include": ["src/**/*"],    // 包含目录
  "exclude": ["node_modules"] // 排除目录
}
```



### 4. 常用指令（全栈必备）

#### 基础编译

```bash
# 按 tsconfig.json 编译整个项目
tsc

# 编译单个文件（忽略配置）
tsc src/index.ts

# 编译并指定输出目录
tsc --outDir dist

# 编译为单个合并文件
tsc src/*.ts --outFile dist/bundle.js
```

#### 开发模式（监听）

```bash
# 监听文件变化，自动编译（热编译）
tsc --watch  # 简写：tsc -w
```

#### 类型检查（不生成文件）

```bash
# 仅做类型检查，不输出 JS（CI/校验常用）
tsc --noEmit

# 仅生成 .d.ts 类型声明文件（库开发）
tsc --declaration --emitDeclarationOnly
```

#### 多环境 / 多配置

```bash
# 指定配置文件（生产/开发分离）
tsc -p tsconfig.prod.json  # --project 简写
```

#### 构建模式（复合项目）

```bash
# 增量构建（大型项目/Monorepo）
tsc --build   # 简写：tsc -b

# 清理构建产物
tsc -b --clean
```



### 5. package.json 脚本（工程化）

```json 
{
  "scripts": {
    "build": "tsc",                    // 生产编译
    "build:prod": "tsc -p tsconfig.prod.json",
    "watch": "tsc --watch",            // 开发监听
    "check": "tsc --noEmit"            // 类型检查
  }
}
```

## 二、nodemon：Node 开发热重载工具

### 1. 核心定位

**开发效率神器**：监听文件变化 → **自动重启应用**，替代 `node` 命令，无需手动重启。

### 2. 安装

```bash
# 本地安装（推荐）
npm install -D nodemon

# 全局安装
npm install -g nodemon

# 验证
nodemon -v
```



### 3. 基础用法（JS 项目）

```bash
# 直接替代 node
nodemon src/index.js

# 传递参数给应用
nodemon src/index.js --port 3000
```



### 4. TS 项目核心用法（配合 ts-node）

**原理**：nodemon 监听 `.ts` → 变化时调用 `ts-node` 直接运行 TS（内存编译，不输出文件）。

#### 安装配套工具

```bash
npm install -D ts-node typescript nodemon
```



#### 命令行直接运行

```bash
# 监听 src 下所有 .ts 文件，变化后用 ts-node 运行
nodemon --watch src -e ts --exec ts-node src/index.ts
```

**参数解析**：

- `--watch src`：只监听 `src` 目录
- `-e ts`：只监控 `.ts` 扩展名
- `--exec`：指定变化后执行的命令



#### 配置文件（nodemon.json，推荐）

```json
{
  "watch": ["src"],
  "ext": "ts",
  "ignore": ["node_modules", "dist"],
  "exec": "ts-node src/index.ts"
}
```

**使用**：直接执行 `nodemon`



#### package.json 脚本（标准）

```json
{
  "scripts": {
    "dev": "nodemon",                // 用配置文件
    "dev:fast": "nodemon --exec ts-node --transpile-only src/index.ts" // 跳过类型检查（更快）
  }
}
```



### 5. 常用指令与参数

#### 核心控制

```bash
# 手动重启（运行时输入）
rs

# 调试模式（配合 Node Inspector）
nodemon --inspect src/index.ts

# 详细日志（排查监听问题）
nodemon --verbose src/index.ts
```



#### 监听规则

```bash
# 监听多个目录
nodemon --watch src --watch config

# 忽略目录
nodemon --ignore test --ignore dist

# 延迟重启（防频繁触发）
nodemon --delay 2 src/index.ts  # 延迟2秒
```



#### 高级执行（TS 全栈常用）

```bash
# 先编译再运行（模拟生产）
nodemon --exec "npm run build && node dist/index.js"

# 使用 tsx（比 ts-node 更快，Rust 编写）
nodemon --exec tsx src/index.ts
```

## 三、tsc vs nodemon：全栈场景分工

### 1. 核心区别

| 工具        | 核心功能                | 阶段     | 输出           | 适用场景                 |
| ----------- | ----------------------- | -------- | -------------- | ------------------------ |
| **tsc**     | TS → JS 编译 + 类型检查 | 编译阶段 | `dist/xxx.js`  | 生产构建、类型校验、打包 |
| **nodemon** | 文件监听 + 自动重启     | 开发阶段 | 无（内存运行） | 开发热重载、调试         |



### 2. 全栈开发标准流程

- **开发环境**：`nodemon + ts-node` → 改代码 → 自动重启 → 即时生效
- **生产环境**：`tsc` 编译 → `node dist/index.js` 运行

## 四、常见问题与避坑

### 1. nodemon 不监听 .ts 文件

- 必须加 `-e ts` 或在 `nodemon.json` 配置 `ext: "ts"`
- 或用 `--files` 参数：`nodemon --exec ts-node --files src/index.ts`

### 2. 热重载慢

- 开发用 `--transpile-only` 跳过类型检查（仅转译）
- 改用 `tsx` 替代 `ts-node`（速度提升 10~20 倍）

### 3. 路径别名（paths）不生效

- 安装 `tsconfig-paths`
- 启动命令：`nodemon --exec ts-node -r tsconfig-paths/register src/index.ts`

### 4. tsc 编译不生成文件

- 检查 `outDir` 配置
- 确认 `include` 包含源码目录
- 无错误时才会输出

## 五、最终推荐（全栈标准配置）

### package.json 最终脚本

```json
{
  "scripts": {
    "dev": "nodemon",                          // 开发（带类型检查）
    "dev:fast": "nodemon --exec tsx src/index.ts", // 极速开发（无检查）
    "build": "tsc",                            // 生产编译
    "start": "node dist/index.js",             // 生产运行
    "check": "tsc --noEmit"                    // 类型校验
  }
}
```



### 配套配置文件

- `tsconfig.json`：编译规则
- `nodemon.json`：监听规则

**一句话总结**：**开发用 nodemon 热重载，生产用 tsc 编译**。
